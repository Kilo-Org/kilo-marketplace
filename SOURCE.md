# PR 149 source trust review

**Scope.** Compared PR head `9c60c49` with base `c7d249097589c6c94808688969097a5048ff5890` on 2026-06-24. Reviewed **88 MCP descriptors** (86 added, 2 materially modified: `github`, `neon`) and **147 added top-level skills** from **93 source repositories**. The marketplace index files were treated as indexes, not separate executable entries. Coverage: **235/235 entries classified; no omissions.**

## Prioritized findings

1. **Do not approve 27 MCPs without publisher/provenance confirmation.** They execute packages or contact services owned by individuals/unrelated organizations; the most sensitive include `privilege-cloud-mcp`, `purview-unified-catalog-mcp`, database MCPs, and npm packages with zero-star/new repositories.
2. **Replace or remove `gitmcp-apache-atlas`.** `gitmcp.io` is an independent service (not Apache/GitHub), is mutable, and intentionally imports public-repository text into the model context; upstream security discussion acknowledges prompt-injection/cross-tool risk.
3. **Pin official artifacts immutably.** Most otherwise credible MCPs use semver/tag pins or mutable remote endpoints without package integrity. Only seven descriptors use a full Git SHA or image digest from an aligned owner.
4. **Remove GitHub MCP legacy fallback.** `github` adds the official `ghcr.io/github/github-mcp-server:1.4.0` image but retains deprecated `@modelcontextprotocol/server-github`; pin the official image by digest.
5. **Constrain placeholder remotes.** `new-relic-ai-mcp`, `oci-database-tools-mcp`, `purview-unified-catalog-mcp`, and `splunk-mcp-server` accept user-supplied hosts. Verify/document an allowlist; New Relic documents `mcp.newrelic.com` / `mcp.eu.newrelic.com`.
6. **Skill provenance is not reproducible.** All 147 skills record repository/path/license but no source commit/tag or content hash. PR contents are fixed by this commit, yet a reviewer cannot prove which upstream revision was imported. Add source commit metadata and automated diff verification.

## Methodology

- Enumerated `git diff --name-status BASE...HEAD`; counted only top-level `skills/*/SKILL.md` as skills and every changed `mcps/*/MCP.yaml` as an MCP descriptor.
- Inspected descriptor source URLs and every runtime package, image, Git URL, and hosted endpoint; distinguished full SHA/digest pins from semver/tag pins and mutable remotes.
- Checked GitHub ownership type, archive state, stars, and latest push with `gh api`; checked selected npm publisher/repository metadata; cross-checked official product documentation for hosted endpoints.
- Classification means: **Trusted** = aligned canonical owner plus immutable runtime reference; **Likely trusted with caveat** = credible/aligned owner but mutable hosted service, semver/tag, beta, proxy, or incomplete provenance; **Unverified/concerning** = weak/misaligned ownership, very new/low-signal source, opaque third-party hosting, or unresolved provenance.
- Repository activity is supporting evidence, not proof. Stars and recent pushes do not establish publisher identity, package integrity, or absence of malicious code.
